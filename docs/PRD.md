# PRD — data-monitor (데이터 모니터링 Tool)

상위 컨텍스트: [../../../docs/PRD.md](../../../docs/PRD.md)

## 목적

`data-persistence`가 저장한 Sample/Order 데이터의 현재 상태를 콘솔에서 실시간으로 조회할 수 있는 독립 관리자 도구. 메인 시스템(`console-mvc`) 내부의 "모니터링" 메뉴와는 별개로, 데이터 저장소 자체를 점검하기 위한 개발/운영 보조 도구다.

## 데이터 소스

`data-persistence`의 Repository(또는 동일 JSON 파일)를 읽기 전용으로 참조한다. 이 도구는 데이터를 변경하지 않는다. 조회 대상 필드는 **[../../../docs/SCHEMA.md](../../../docs/SCHEMA.md)** 를 따른다.

## 기능 요구사항

1. **전체 요약**: 등록 시료 수, 전체 주문 수, 상태별 주문 건수(`RESERVED`/`REJECTED`/`PRODUCING`/`CONFIRMED`/`RELEASE`) — `REJECTED` 포함은 의도된 사항이다(결정 완료, 통합 저장소 [docs/DECISIONS.md](../../../docs/DECISIONS.md) 참조). `console-mvc`의 메인 콘솔 모니터링 메뉴만 `REJECTED`를 제외한다.
2. **시료 목록 조회**: 전체 Sample 레코드와 현재 재고, 그리고 재고 상태 분류(여유/부족/고갈)를 표 형태로 출력 (결정 완료 — 통합 저장소 [docs/DECISIONS.md](../../../docs/DECISIONS.md) 참조)
   - `console-mvc`의 모니터링 메뉴와 달리 이 도구의 분류는 주문 수요 대비 상대값이 아니라 **절대 재고 수량 기준**의 2단계 임계값으로 판정한다: `plenty_threshold`(기본값 1000) 이상이면 "여유", `shortage_threshold`(기본값 300) 이상 `plenty_threshold` 미만이면 "부족", `shortage_threshold` 미만(0 포함)이면 "고갈".
   - 두 임계값은 하드코딩된 상수가 아니라 콘솔 명령으로 실행 중 변경 가능해야 한다 (예: 별도 메뉴 또는 명령 인자로 `plenty_threshold`/`shortage_threshold` 조회·수정).
3. **주문 목록 조회**: 전체 Order 레코드를 상태별 필터링하여 출력 (예: 상태 선택 후 해당 목록만 표시)
4. **실시간 갱신**: 재조회 명령(예: 새로고침 입력) 시 파일을 다시 읽어 최신 상태 반영 — 상주 프로세스가 자동 폴링할 필요는 없으며, 사용자가 명시적으로 갱신을 트리거하는 방식으로 충분

## 실행 형태

독립 실행 가능한 콘솔 앱(별도 entry point)으로, 메인 시스템과 동시에 실행되어도 같은 JSON 파일을 안전하게 읽을 수 있어야 한다 (읽기 전용이므로 동시성 이슈는 최소화됨).

## 완료 기준

- 메인 시스템이나 `dummy-data-generator`로 데이터가 추가/변경된 후, 이 도구를 재조회하면 변경 사항이 반영되어 보인다.
- 이 도구 실행/조회로 인해 원본 데이터가 변경되지 않는다.
- 재고 상태(여유/부족/고갈)가 기본 임계값(1000/300)으로 정확히 분류되고, 콘솔에서 임계값을 변경하면 이후 조회에 즉시 반영된다.
